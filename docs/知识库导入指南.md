## 支持的数据格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Markdown | `.md` | 按标题层级拆分内容 |
| JSON | `.json` | 预结构化文档（推荐） |
| 纯文本 | `.txt` | 按字数分块 |
| Word | `.docx` | 支持标题、段落、表格、列表 |
| PowerPoint | `.pptx` | 按幻灯片拆分 |
| PDF | `.pdf` | 按页面拆分，支持表格提取 |
| Excel | `.xlsx/.xls` | 支持多 Sheet |

## 格式说明

### 1. JSON 格式（推荐）

JSON 格式最灵活，支持完整的元数据配置。

**文件结构：**

```json
{
  "title": "文档标题",
  "domain": "领域名称",
  "doc_type": "knowledge",
  "sections": [
    {
      "heading": "章节标题",
      "content": "章节内容",
      "tags": ["标签1", "标签2"]
    },
    {
      "heading": "另一个章节",
      "content": "更多内容...",
      "tags": ["标签3"],
      "domain": "覆盖默认领域",
      "doc_type": "覆盖默认类型"
    }
  ]
}
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 否 | 文档标题 |
| `domain` | 否 | 领域标签（默认 `general`） |
| `doc_type` | 否 | 文档类型（`knowledge`/`rule`/`case`，默认 `knowledge`） |
| `sections` | 是 | 章节数组 |
| `sections[].heading` | 否 | 章节标题 |
| `sections[].content` | 是 | 章节内容 |
| `sections[].tags` | 否 | 标签数组 |
| `sections[].domain` | 否 | 章节级领域覆盖 |
| `sections[].doc_type` | 否 | 章节级类型覆盖 |

### 2. Markdown 格式

Markdown 会按标题层级自动拆分章节。

**示例：**

```markdown
# 一级标题

这是一级标题下的内容。

## 二级标题

这是二级标题下的内容。

### 三级标题

这是三级标题下的内容。
```

### 3. Word 文档 (.docx)

- 标题自动识别为 Heading 1-9 → #-######
- 表格自动转换为 Markdown 表格
- 列表自动识别
- 图片位置会被标记（当前版本不提取图片内容）

### 4. PowerPoint 文档 (.pptx)

- 按幻灯片拆分
- 幻灯片标题提取为 # 标题
- 内容提取为正文
- 备注提取为 > **备注**: ...

### 5. PDF 文档 (.pdf)

- 按页面拆分
- 支持表格提取
- 优先使用 `pdfplumber`（MIT 许可证）
- 备选使用 `PyMuPDF`（速度更快）

### 6. Excel 文档 (.xlsx/.xls)

- 支持多 Sheet
- 每个 Sheet 转换为 Markdown 表格
- Sheet 名称作为二级标题

## 入库命令

### 方式一：通过 ingest.py 直接入库

```bash
# 入库单个文件
python -m knowledge.pipelines.ingest --file path/to/document.docx --domain marketing

# 入库整个目录
python -m knowledge.pipelines.ingest --dir knowledge/raw --domain marketing

# 入库并保存 Markdown 中间文件
python -m knowledge.pipelines.ingest --dir knowledge/raw --output-markdown --markdown-output-dir knowledge/parsed

# 指定文档类型
python -m knowledge.pipelines.ingest --file rules.docx --doc-type rule

# 查看知识库统计
python -m knowledge.pipelines.ingest --stats
```

### 方式二：通过 main.py 入库

```bash
# 入库目录
python main.py --ingest knowledge/raw --ingest-domain marketing

# 入库并保存 Markdown
python main.py --ingest knowledge/raw --output-markdown --markdown-output-dir knowledge/parsed

# 查看统计
python main.py --stats
```

### 命令参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--file FILE` | 入库单个文件 | - |
| `--dir DIR` | 入库整个目录 | - |
| `--stats` | 查看知识库统计 | - |
| `--domain DOMAIN` | 领域标签 | `general` |
| `--doc-type DOC_TYPE` | 文档类型 | `knowledge` |
| `--output-markdown` | 保存 Markdown 中间文件 | `False` |
| `--markdown-output-dir DIR` | Markdown 输出目录 | `knowledge/parsed` |

## 知识库分类

知识库分为三个独立的 Collection，通过 `doc_type` 区分：

| Collection | doc_type | 用途 |
|------------|----------|------|
| `kb_knowledge` | `knowledge` | 通用知识、FAQ、术语解释、文档片段 |
| `kb_rules` | `rule` | 规则、约束、SOP、审批流程 |
| `kb_cases` | `case` | 案例、复盘、决策记录 |

**使用建议：**

- **knowledge**：产品文档、用户手册、常见问题
- **rules**：公司制度、审批流程、业务规则
- **cases**：成功案例、失败复盘、历史决策
